# Version Management Guide

## Current Version
See `VERSION` file for current version number.

## Semantic Versioning

StrawGo follows [Semantic Versioning 2.0.0](https://semver.org/)

### Version Format: MAJOR.MINOR.PATCH

- **0.x.x** - Pre-1.0 releases (Alpha/Beta)
  - Breaking changes may occur between minor versions
  - 0.1.0 → 0.2.0 may have breaking changes

- **1.0.0+** - Stable releases
  - **MAJOR** - Incompatible API changes
  - **MINOR** - New features (backward compatible)
  - **PATCH** - Bug fixes (backward compatible)

## How to Update Version

### 1. Bug Fix (0.0.2 → 0.0.3)
```bash
echo "0.0.3" > VERSION
# Update CHANGELOG.md with bug fixes
```

### 2. New Feature (0.0.3 → 0.0.4)
```bash
echo "0.0.4" > VERSION
# Update CHANGELOG.md with new features
```

### 3. When ready for beta (0.0.x → 0.1.0)
```bash
echo "0.1.0" > VERSION
# Update CHANGELOG.md
# Mark as beta release
```

### 4. First Stable Release (0.x.x → 1.0.0)
```bash
echo "1.0.0" > VERSION
# Update CHANGELOG.md
# Announce stable release
```

## Files to Update

When releasing a new version:

1. **VERSION** - Single line with version number
2. **CHANGELOG.md** - Add new version section at top
3. **IMPLEMENTATION_COMPLETE.md** - Update version field
4. **src/processors/aggregators/README.md** - Update version

## Git Tagging

```bash
# After updating VERSION and CHANGELOG
git add VERSION CHANGELOG.md
git commit -m "Bump version to X.Y.Z"
git tag -a vX.Y.Z -m "Version X.Y.Z"
git push origin main --tags
```

## Release Checklist

- [ ] Documentation updated
